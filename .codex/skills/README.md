# Repo Skills

Put repo specific skills in this folder.

Use this path shape:

`.codex/skills/<skill-name>/SKILL.md`

Rules:

- Keep each skill focused on one repeatable task.
- Keep each skill short.
- Put examples, prompts, or helper files in the same skill folder.
- Use clear names like `supabase-rls`, `meilisearch-sync`, or `kamal-release`.

You can copy `.codex/_template/skill.md` to start a new skill.
